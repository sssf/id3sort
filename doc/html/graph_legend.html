Grafförklaring

Den här sidan förklarar hur man ska tolka de grafer som doxygen genererar.

Tag följande exempel:

/*! Osynlig klass på grund av stympning */
class Invisible { };
/*! Stympad klass, ärvningsrelationen är dold */
class Truncated : public Invisible { };
/* Klass utan doxygen-kommentarer */
class Undocumented { };
/*! Klass som ärvs med publikt arv */
class PublicBase : public Truncated { };
/*! En template-klass */
template<class T> class Templ { };
/*! Klass som ärvs med skyddat arv */
class ProtectedBase { };
/*! Klass som ärvs med privat arv */
class PrivateBase { };
/*! Klass som används av Inherited klassen */
class Used { };
/*! Super klassen som ärver ett antal andra klasser */
class Inherited : public PublicBase,
protected ProtectedBase,
private PrivateBase,
public Undocumented,
public Templ<int>
{
private:
Used *m_usedClass;
};

Rektanglarna i den ovanstående grafen har följande betydelser:

  • En fylld svart rektangel representerar den strukt eller klass som har genererat grafen.
  • En rektangel med svart kant symboliserar en dokumenterad strukt eller klass.
  • En rektangel med grå kant symboliserar en odokumenterad strukt eller klass.
  • En klass med röd kant symboliserar en strukt eller klass där alla dess relationer inte visas. En graf stympas om den inte får plats inom de givna gränserna.

Pilarna har följande betydelser:

  • En mörkblå pil används för att visualisera en publik arvrelation mellan två klasser.
  • En mörkgrön pil används för en skyddad arvsrelation.
  • En mörkröd pil används för en privat arvsrelation.
  • En sträckad lila pil används om en klass är innesluten eller använd av en annan klass. Vid pilen står namnet på den eller de variabler som klassen pilen pekar på kommer åt.
  • En sträckad gul pil symboliserar förhållandet mellan en template-instans och template-klassen, som den instantierades från. Vid pilen står instansens template-parametrar.
